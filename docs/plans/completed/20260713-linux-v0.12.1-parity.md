# Linux v0.12.1 parity plan

## Objective

Bring `linux-port` from the published Linux v0.11.0 release to upstream tag `v0.12.1`.
Preserve upstream protocol and model behavior while providing native GTK4/libadwaita equivalents for every applicable user-visible change.

## Upstream range

Included:

- Upstream `v0.11.0..v0.12.1` (`450ac16..d29b64a`).
- Sidebar directory drops and session-folder reveal from #180.
- Codex lifecycle-driven agent status from #194 and its v0.12.1 auto-review correction.
- Split-survivor promotion and the split-focus retry correction from #121 and #200.
- Dashboard grid overlay, its GUI/keymap/control surfaces, and its v0.12.1 titlebar correction from #202 and #205.
- Spawned-shell `TERM_PROGRAM` identity correction from #203.
- Shared documentation, bundled agent skill, tests, and release notes already present in upstream.

Excluded or adapted:

- macOS Finder and AppKit mechanics are replaced with the default Linux file manager and GTK drag/drop.
- macOS titlebar and traffic-light behavior is replaced with libadwaita header-bar behavior.
- XCUITests cannot run on Linux; equivalent host-free, CLI, GTK runtime, and AT-SPI coverage is used where practical.
- Wayland window placement remains compositor-owned and is unrelated to this range.
- `site/_screenshot.png` and other pre-existing untracked artifacts remain excluded.

## Integration rules

- Keep upstream `agtermCore/` files unchanged on the downstream side.
- Never modify `agtermCore/Tests/agtermCoreTests/ConfigPathsTests.swift` downstream.
- Keep GTK, Glibc, Linux CLI, packaging, and runtime adapters in Linux-owned paths.
- Reuse upstream dashboard, control-dispatch, recency, hook-merge, and skill-install decisions rather than copying them.
- Keep GUI, keymap, control socket, Linux CLI, read-back, bundled skill, package contents, and documentation synchronized.
- Use the GTK action palette where upstream uses macOS application menus.

# Linux v0.8.2 parity plan

## Overview

Bring `linux-port` to upstream agterm behavior parity through upstream tag `v0.8.2`.
Linux releases will follow upstream version numbering, so this plan is the gate for a Linux `v0.8.2` release.

Success criteria:

- Every user-visible upstream behavior change from 2026-06-30 through `v0.8.2` is either implemented on Linux, verified already present, or explicitly marked not applicable.
- Linux-only code stays inside `agterm-linux/`, `linux/`, `packaging/linux/`, or Linux scripts.
- No Glibc/Linux compatibility code is added back to upstream-owned macOS/global files unless it is intentionally portable and acceptable upstream.
- The Linux app and Linux-local `agtermctl` build and run.
- Control API, CLI help, README, and bundled agent-skill docs agree for Linux-relevant behavior.

## Context

Upstream tags covered:

- `v0.4.0` through `v0.8.2`.
- `v0.8.1` and `v0.8.2` are macOS microphone permission fixes.
- The post-`v0.8.2` commit `f007efe` is intentionally out of scope for this plan and should be handled in the next version slice.

Linux boundary files:

- Linux UI/controller: `agterm-linux/Sources/AgtermLinux/`.
- Linux-local CLI: `agterm-linux/Sources/agtermctl/`.
- Shared model/protocol already consumed by Linux: `agtermCore/Sources/agtermCore/`.
- Linux scripts: `scripts/*-linux.sh`, `scripts/dist-linux.sh`.
- Linux release docs: `README.md`, `AGENTS.md`.

Important current known gaps from the audit:

- Floating overlay still auto-selects its target instead of using upstream `--follow` semantics.
- Workspace expansion/collapse is Linux-local view state and is not persisted through `Workspace.isExpanded`.
- GUI new-session directory still defaults to home instead of the configured `AppSettings.newSessionDirectory` policy.
- Per-session close confirmation setting is not wired for Linux GUI close-session actions.
- `window.zoom` returns unsupported; decide whether GTK maximize/unmaximize is acceptable parity.

## Version-by-version gap matrix

### v0.4.0

Implemented or already present on Linux:

- Repeated status sound debounce via `SoundThrottle`.
- File/text drops insert escaped text into the terminal.
- Right/middle mouse buttons are forwarded to libghostty.
- Split open focuses the new pane.
- `session.resize` exists in the Linux CLI/control path.

Needs verification or small fix:

- Esc during inline rename should cancel the rename and restore terminal focus.
- Fish shell agent-status hook support depends on whether the Linux release installs the bundled hooks; verify docs/scripts if included.

Not applicable:

- macOS cask metadata.

### v0.4.1

Implemented or already present on Linux:

- Restore-running-command support exists through `/proc/<pid>/cmdline`, `restore-denylist.conf`, and quit capture.
- Right-click paste default and setting exist in Linux Settings.
- File-drop targeting has Linux surface guards for sidebar drag payloads.

Needs verification:

- Restore-running-command should be smoke-tested for primary and split panes on Linux before release.

### v0.5.0 to v0.5.2

Implemented or already present on Linux:

- `session.background` image/text/color through shared `BackgroundWatermark` and Linux watermark rendering.
- Toolbar/sidebar shortcut hints via shared keymap defaults where Linux exposes tooltips.
- `session.text` reads terminal buffer text.
- Unseen notification badge/pill exists in the Linux sidebar.
- Split toolbar icon active-state variants exist in Linux resources.
- Per-overlay background color exists.

Needs implementation:

- Configurable new-session directory from `AppSettings.newSessionDirectory` is not used by Linux GUI new-session actions.

Needs verification:

- Sidebar scrollbar behavior when content fits.
- Split toolbar icon states for no split / split shown / collapsed left / collapsed right.

Not applicable:

- macOS Dock badge; Linux row badge is the relevant equivalent.

### v0.6.0 to v0.6.1

Implemented or already present on Linux:

- Keymap store/config loading.
- Palette catalog/fuzzy ranking/theme catalog hoists consumed by Linux.
- Overlay capture via `OverlayCapture`.
- Custom command matcher.
- Shared control resolve/mode helpers.
- Shell escape and custom-command quoting fixes.
- Bracketed paste/drop safety.
- OSC title/pwd sanitization.
- OSC 52 clipboard gating.
- Scratch hidden under full overlay.
- Pane-addressable `session.type` / `session.text`, including `scratch`.
- Wrapped session next/prev navigation.

Needs implementation:

- Per-session close confirmation setting for Linux GUI close-session actions.

Needs verification:

- Scratch-under-full-overlay behavior after recent overlay changes.
- OSC 52 ask/deny/write behavior under GTK clipboard APIs.

### v0.7.0 to v0.7.1

Implemented or already present on Linux:

- Auto-follow attention and settings controls.
- Ctrl-Tab MRU switcher path exists; verify cap at 10 sessions.
- Per-call `session.status --color`.
- Pane-aware agent status and pane reveal behavior.
- Link hover cursor and validated link opening.
- Window controls mostly exist in Linux control/CLI path.

Needs implementation:

- Workspace expansion/collapse persistence.
- `window.zoom` parity decision and possible GTK maximize/unmaximize implementation.

Needs verification:

- Full-row workspace collapse click behavior.
- Auto-follow settings text and behavior parity.

Not applicable:

- About panel link to agterm.com unless Linux has an About dialog.

### v0.8.0

Implemented or already present on Linux:

- `session new --after/--before` and `session move --after/--before` exist in Linux-local CLI and dispatcher.
- Overlay background color exists.
- Window lifecycle commands exist in Linux CLI, with Wayland limitations for move/zoom noted.

Needs implementation:

- Overlay `--follow` semantics:
  - add protocol field `follow`,
  - add Linux-local CLI flag,
  - pass through `LinuxControlDispatcher`,
  - select only when `follow` is true,
  - ensure full and floating overlays open in background by default.
- Workspace expand/collapse persistence is part of this release slice.

Structural parity note:

- Upstream moved remaining commands into `ControlDispatcher`.
  Linux intentionally keeps a Linux-local synchronous dispatcher to preserve GTK/C callback boundaries.
  Mirror upstream validation and response shape, but do not reintroduce Linux sync requirements into `agtermCore`.

### v0.8.1 and v0.8.2

Not applicable to Linux release behavior:

- `NSMicrophoneUsageDescription` and `com.apple.security.device.audio-input` are macOS TCC/hardened-runtime requirements.
- Linux has no equivalent app entitlement in this packaging model.

Release note guidance:

- Do not list these as Linux fixes unless Linux later adds a portal/sandbox microphone permission model for Flatpak or another package format.

## Development Approach

Use small, reviewable tasks in version order.
Use regular implementation with focused tests rather than a large rewrite.
Each task must include at least one code-level validation and one runtime/control smoke when behavior is user-visible.
Keep Linux-only logic inside `agterm-linux/` or scripts.
Do not update `CHANGELOG.md` until the Linux release flow.

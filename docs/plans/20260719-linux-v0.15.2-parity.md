# Plan: Linux v0.15.2 parity and release

Bring `linux-port` to the exact upstream `v0.15.2` release, adapt every portable product change to the
GTK4/libadwaita host, validate the Linux application and packages, and publish `linux-v0.15.2` only after the
required push, tag, and release approval gates.

The selected boundary is upstream tag `v0.15.2` at `95926fc88fd7c6e7f3e1acf229cb2dab637fdbe0`.
Exclude upstream `master` commits `8220978` and `4075371`, which are newer than that tag.
Local `master` is exact at the tag and has already been pushed with approval.
Local `linux-port` contains the unpushed merge commit `90147b8`.

The following constraints apply throughout:

- Keep shared controller, protocol, persistence, and catalog behavior host-free and upstream-compatible.
- Keep GTK/libadwaita, Linux notification, process, window, rendering, and packaging adaptations under Linux-owned roots.
- Never edit `agtermCore/Tests/agtermCoreTests/ConfigPathsTests.swift` downstream.
- Accept upstream's tagged `CHANGELOG.md` verbatim and add no downstream entries.
- Keep GUI, keymap, palette, and control-socket surfaces in sync where an action is controllable.
- Leave `site/_screenshot.png` and unrelated user state untouched.
- Stop for approval before pushing `linux-port`, creating or pushing `linux-v0.15.2`, or publishing the release.

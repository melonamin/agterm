# Linux v0.11.0 parity plan

## Objective

Bring `linux-port` from its completed v0.8.2 parity work to upstream tag `v0.11.0`.
Do not cut an intermediate v0.8.2 Linux release.

The final Linux branch should preserve upstream shared behavior and protocol shapes while implementing GTK4/libadwaita equivalents for all applicable user-visible features.

## Scope

Included:

- Upstream `v0.8.2..v0.11.0`.
- Shared `agtermCore` and `agtermctlKit` changes from upstream.
- Linux GTK UI, control adapter, CLI, documentation, packaging, and tests needed for parity.
- Portable socket support merged upstream through PR #151.

Excluded:

- Post-v0.11.0 commits `a5ef491` / PR #180 and `b00362f` / PR #194.
- macOS-only Finder, AppKit, titlebar, traffic-light, TCC, entitlement, cask, and notarization mechanics.
- `CHANGELOG.md` changes outside the Linux release flow.

## Integration rules

- Treat upstream `agtermCore/` and shared CLI contracts as authoritative.
- Resolve shared-code conflicts toward upstream unless a portable Linux fix is deliberately being proposed upstream.
- Keep GTK, libadwaita, Glibc adapters, and Linux packaging inside Linux-owned paths.
- Preserve surface ownership and copy C strings before crossing callback or queue boundaries.
- Keep GUI, keymap, control socket, CLI, bundled agent skill, and documentation behavior synchronized.
- Keep `site/_screenshot.png` outside commits unless it is explicitly selected as a documentation asset.

# Linux v0.13.0 parity plan

## Objective

Merge upstream `v0.13.0` into `linux-port` and adapt every applicable product change to GTK4/libadwaita.
Keep shared code identical to upstream and place all host behavior in the Linux application and CLI targets.

## Upstream inventory and Linux adaptation

- Add title-bar recent-session and attention popovers using native themed GTK popovers.
- Add live agent-status glyphs to dashboard captions using the existing Linux status colors and blink behavior.
- Set `$AGT_PANE` correctly for left, right, and scratch custom-command origins.
- Generate `$AGTERM_PANE_ID` per terminal surface and resolve status updates against the surface's current pane role.
- Forward `--pane-id` from the Linux `agtermctl session status` command.
- Honor libghostty mouse visibility, link-hover, and cursor-shape actions with GTK cursor names.
- Keep the v0.13.0 title-bar action cluster together at the content-header end.
- Omit Dock-icon bouncing on Linux because Wayland/X11 desktop shells expose no portable application bounce API.
  Linux desktop notifications, unseen badges, attention status, and popovers remain supported.

The popovers are control-API exempt because opening transient mouse UI has no meaningful headless operation.
Session selection and status routing remain available through the existing control commands.
